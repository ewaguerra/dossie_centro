# Acessibilidade visual global

Fonte: `vendor/app/styles/a11y.css`  
Carregamento: **segundo** `<link>` após `tokens.css` nas 4 páginas principais.

## Foco visível (`:focus-visible`)

Aplica outline âmbar (`--color-accent`) a:

- `button`, `[role="button"]`, `a`, `input`, `select`, `textarea`, `summary`
- `[tabindex]:not([tabindex="-1"])`
- Componentes DS: `.btn`, `.nav-retorno-link`, `.nav-btn`, `.hamburger-*`

Propriedades: `2px solid`, `outline-offset: 2px`, `border-radius: var(--radius-xs)`.

**Override Centro:** `.has-jesuit-frame` usa `--color-accent-strong` (vermelho HUD) em botões ops da sidebar/nav.

Estilos locais de foco **mantidos** onde acrescentam contexto (ex.: `.clue-word:focus-visible` em arquivo-morto).

## Reduced motion

`@media (prefers-reduced-motion: reduce)`:

- Desativa animações/transições globalmente
- `scroll-behavior: auto` em `html`
- Regras locais (landing scanlines, centro `.as-shake`) complementam o global

## Validação

```bash
grep -R "focus-visible" vendor landing centro arquivo-morto arquivista
grep -R "prefers-reduced-motion" vendor landing centro arquivo-morto arquivista
npm test   # HTTP 200 /app/styles/a11y.css + sanity
```

Manual: Tab/Shift+Tab nas 4 páginas; DevTools → Rendering → `prefers-reduced-motion: reduce`.

## Limitações

- Não substitui auditoria WCAG completa (`docs/capri/wcag-contrast-notes.md`).
- Elementos custom sem `button`/`a`/tabindex podem precisar de regra local.
